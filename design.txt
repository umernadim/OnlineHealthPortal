# 🌐 Online Health Portal – Frontend Design Guide

## 🔹 Required Pages & Components

### 1. **Landing / Home Page**
- **Components:**
  - Header (Logo + Navigation bar: Home, Doctors, Login, Signup)
  - Hero section (Banner: “Consult Doctors Online” + CTA button)
  - Quick links (Book Appointment, Upload Records, View Doctors)
  - Testimonials / Reviews section
  - Footer (Contact info, social links, privacy policy)

---

### 2. **Login / Signup Page**
- **Components:**
  - Role selection (Patient / Doctor / Admin)
  - Login form (Email, Password)
  - Signup form (Name, Email, Password, Role, Specialty for doctors)
  - Forgot password option

---

### 3. **Patient Dashboard**
- **Components:**
  - Sidebar (Appointments, Records, Prescriptions, Notifications)
  - Upcoming appointments list
  - Quick action buttons (Book Doctor, Upload Report, View Prescription)
  - Notifications panel (Reminders, test results)

---

### 4. **Doctor Dashboard**
- **Components:**
  - Sidebar (Patient list, Appointments, Prescriptions, Notifications)
  - Patient profile view (Medical history, uploaded reports)
  - Appointment schedule (Calendar view)
  - Prescription writing form
  - Follow-up scheduling option

---

### 5. **Admin Dashboard**
- **Components:**
  - Manage doctors (Add, Edit, Approve profiles)
  - Manage patients (View records, reset accounts)
  - Appointment tracking (List + status)
  - Reports & analytics (basic charts: active users, appointments per week)

---

### 6. **Doctor Listing / Search Page**
- **Components:**
  - Search bar (by name, specialty, language)
  - Filters (Specialty, Availability, Ratings)
  - Doctor cards (Photo, Name, Specialty, Experience, Ratings, “Book Now” button)

---

### 7. **Appointment Booking Page**
- **Components:**
  - Doctor profile summary
  - Calendar & time slot selector
  - Consultation mode (Video / Audio / Chat)
  - Confirm appointment button
  - Appointment summary card

---

### 8. **Video Consultation Page**
- **Components:**
  - Video call window
  - Chat sidebar (for notes/questions)
  - Screen sharing option
  - End call button
  - Prescription upload option (doctor side)

---

### 9. **Patient Health Records Page**
- **Components:**
  - Upload button (PDF, Images, Reports)
  - Records list (Prescriptions, Lab reports, X-rays)
  - Secure access (only patient + assigned doctor)
  - Download option

---

### 10. **Prescription Page**
- **Components:**
  - Doctor prescription view (Medicine name, dosage, instructions)
  - Patient view (Download / Print option)
  - Medicine reminder toggle
  - Follow-up appointment button

---

### 11. **Chat Page**
- **Components:**
  - Chat window (Patient ↔ Doctor)
  - Message input box
  - File attachment option (reports/images)
  - Status indicator (online/offline doctor)

---

### 12. **Notifications Page**
- **Components:**
  - Appointment reminders
  - Medicine reminders
  - Test result updates
  - Push notification toggle (on/off)

---

### 13. **Invoice / Payment Page (Optional)**
- **Components:**
  - Invoice summary (Doctor fee, consultation type, date)
  - Insurance details form
  - Download invoice button
  - No real payment gateway (just mock bill)

---

## 🔹 UI/UX Flow (Simplified)
1. **Home → Login/Signup → Dashboard (role-based)**
2. **Patient → Doctor listing → Appointment booking → Video consultation → Prescription → Records**
3. **Doctor → Dashboard → Patient records → Prescription → Follow-up**
4. **Admin → Dashboard → Manage doctors/patients → Analytics**

---

## 🔹 Design Tips
- Use **clean, minimal UI** (healthcare = trust + clarity).
- Colors: **Blue + White + Green** (professional + healthcare vibe).
- Icons: Use medical icons (stethoscope, pill, calendar).
- Responsive design (mobile-first approach).
- Keep **forms short & simple** (avoid overwhelming users).
